## TL;DR

上一個版本的 `top: random(0, 100)%` 有兩個不直覺的點:

- 百分比基準是按鈕大小 — 想知道「粒子離中心多遠」要先把按鈕尺寸算進去。
- `0%` 在左上角而不是中心,要靠 `transform: translate(-50%, -50%)` 把錨點推到中心。

重構成:

- 粒子預設用 `inset: 0; margin: auto` 自動置中(粒子尺寸固定,不再需要 `transform: translate(-50%, -50%)` 把戲)。
- 隨機位移用 px:`x = random(-MAGNITUDE, MAGNITUDE)`,套到 `transform: translate(x, y)`。
- Keyframe 改成 `from { transform: translate(0, 0) }`,`to` 留給 inline style 補。

## Why it works

`inset: 0; margin: auto` 是經典置中招(只在 `position: absolute` + 已知尺寸時生效)。粒子 12×12,按鈕 64×64,自動 margin 把粒子放中心。

```ts
const MAGNITUDE = 48;
const x = random(-MAGNITUDE, MAGNITUDE);
const y = random(-MAGNITUDE, MAGNITUDE);
particle.style.transform = `translate(${x}px, ${y}px)`;
```

`MAGNITUDE: 48` 直接表達「最多飛 48px」 — 想要更狂就調大,根本不用算百分比反推。

## Gotchas

- **`transform` 一條被佔走**:partial keyframe 跟 inline style 兩邊都在用 `transform`。如果你還想加 `transform: scale(...)` 之類的,會跟 `translate` 互相覆蓋。要嘛用 CSS 變數(`translate(var(--x), var(--y)) scale(1.2)`),要嘛切到 `translate` 屬性(獨立於 `transform`,新瀏覽器才支援)。
- **CSS Grid 是替代解**:`display: grid; place-items: center` 也能置中,但會把按鈕內所有元素都疊在 `grid-area: 1 / 1`,連 `<svg>` heart 也被當成 grid item — 對既有 children 太強勢,Comeau 偏好 auto-margins。
- **重點不是省哪幾行 code**:這個重構的核心是「**心智模型從相對轉絕對**」 — 之後做粒子 effect 時,腦子裡會想「想飛 60px 半徑」而不是「想飛 90% 容器寬」。
