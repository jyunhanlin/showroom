## TL;DR

之前 demo 都是「同一個 DOM 節點移到新位置」。但 VT 的 contract 其實只要求 **同一輪 view 裡 view-transition-name 是 unique**。

新舊兩端是不是同一個 DOM 節點?VT **不在乎**。

經典案例:nav 的 active underline。`<a class="active">` 上的 `::after` pseudo-element 是 active class 出現時才被瀏覽器建立、消失時銷毀的。每次 active 切到別的 link,**新舊 underline 不是同一個 element**(連 host 都不同)— 但只要兩個都帶 `view-transition-name: underline`,VT 把它們當同一個 morph,自動算 matrix transform 滑過去。

## 寫法

```css
.nav-link.active::after {
  content: '';
  view-transition-name: underline;
  /* underline 樣式 */
}
```

```ts
function handleClick(target) {
  function update() {
    document.querySelector('.nav-link.active')?.classList.remove('active');
    target.classList.add('active');
  }
  document.startViewTransition?.(update) ?? update();
}
```

view-transition-name 同名(`underline`)是身份標記。每輪 view 只能有一個元素帶這個名字 — 滿足這條件即可。

## 跨節點的隱含好處

- **不需要 JS 算位置**:傳統做法是用 `getBoundingClientRect()` 算 active link 的位置,然後 transform underline。VT 自動。
- **後續同步問題消失**:JS 寫法在 resize / font swap / 新增 link 時要重算。VT 每次 transition 都重新拍 snapshot,免重算。
- **跨頁面 VT 必走這條路** ([下一篇](./07.08-vt-route-transitions)):兩個頁面的 hero image 是不同 DOM 節點,但共用 vt-name 就能無縫 morph。

## 常見坑:不同形狀 morph

兩端 element 高/寬比例不同時,瀏覽器補間 size 跟 position,但 cross-fade 期間會看到怪怪的扭曲(高的那端在縮、寬的那端在拉)。

修法:固定 old/new 的尺寸:

```css
::view-transition-old(underline),
::view-transition-new(underline) {
  height: 3px;        /* 強制底線高度固定 */
}
```

讓尺寸內插的維度只剩寬度跟位置,看起來乾淨多了。

## 用 motion layout 還是 VT

motion 的 `layout` prop 解的是同一類問題(LayoutId 跨元素 morph)。

| | motion `layoutId` | VT `view-transition-name` |
|---|---|---|
| Interrupt | 平滑(spring 串接) | 突兀(fast-forward) |
| 跨頁面 | 不行(SPA 內) | 行(via @view-transition) |
| 複雜 layout 變動 | 強(內建 FLIP 多年) | 新但快進步 |
| Bundle | motion ~30kb | 內建瀏覽器 |
| 瀏覽器支援 | 廣 | Chrome / Safari / FF (90%+) |

通常專案用 React + 已裝 motion → 用 motion layout 更穩。沒有 React 或要做 cross-document → VT 是唯一解。

## Gotchas

- **同一輪 view 內名字不能撞**:active class 切換是 atomic 的(remove + add 在同一個 callback),所以「兩個元素都 active」這個 transient state 不存在。只要 callback 內部完成切換就 OK。
- **要 unique key**:active state 是「這個 tab 是 active」,不是「這個 li 是 active」。如果你寫條件 render 把 active link 放別的位置(不同 parent),VT 仍能 morph。
- **`::after` 是常用 host**:underline / pill highlight 不需要新 div,直接 `.active::after { view-transition-name: ... }` 就好。創新元素反而碰到 layout 重排問題。
- **跨層級 morph 要小心 stacking**:從 sidebar 的 active 跳到 header 的 active,兩個位置 z-index 不同。VT 會把 underline 暫時提到 ::view-transition layer(最上層),動畫期間都是可見的;結束後跳回真實 DOM 的 z-index。
- **Server-rendered active state**:initial render 時瀏覽器不會 trigger VT(沒有 startViewTransition call),只有 client-side 的 active 切換才 morph。第一次 active 是 instant 的不要被嚇到。
