## TL;DR

- `animation-timeline` 把 keyframe 動畫的「時間軸」從 wall-clock 換成 scroll 進度。`scroll()` 綁 scroll 容器的 0–100% 進度、`view()` 綁元素「穿過 viewport 的進度」。寫法是純 CSS，不用 JS、不用 IntersectionObserver。
- `animation-range` 決定要把 keyframe 的 0–100% 對應到哪段 scroll 範圍：`cover`（元素第一個 pixel 進入到最後一個 pixel 離開）、`contain`（完全在 viewport 內那段）、`entry`（進場那段）、`exit`（出場那段）。也能自訂 `entry 0% entry 150%` 這種百分比。
- `animation-fill-mode: both` 幾乎是預設要寫的 — view timeline 在 range 之外 keyframe 不生效，沒 fill mode 元素會在 entry 之前彈回原樣、exit 之後也彈回原樣，有明顯 flicker。
- Inline 元素的 `view()` 抓的是**容器**的 view progress 不是自己的 — 字級 stagger 必須 `display: inline-block` 每個 word 才有獨立的 view-timeline。
- 命名 timeline (`view-timeline: --name`) + `timeline-scope: --name` 可以讓 A 元素的動畫被 B 元素的 view progress 驅動 — 跨元素 timeline 連結。
- 瀏覽器支援：Chrome/Edge/Safari TP 有，Firefox 沒。production 用 [scroll-timeline-polyfill](https://github.com/flackr/scroll-timeline) — 但 polyfill 不支援所有 advanced features（命名 timeline 部分支援）。

## `scroll()` vs `view()`

兩種 timeline 解決的是不同問題：

- **`scroll()`**：要追的是「整個 scroll 容器的進度」。reading progress bar、scroll-linked parallax、章節大綱進度標記都是這類。
- **`view()`**：要追的是「這個元素自己穿過 viewport 的進度」。fade-in on enter、parallax-on-image、卡片 reveal 都是這類。

```css
/* 容器 scroll 0% → 100% 對應到 keyframe 0% → 100% */
.box {
  animation: spin linear;
  animation-timeline: scroll();
}

/* 元素自己穿過 viewport 的進度對應到 keyframe */
.card {
  animation: slide-in linear both;
  animation-timeline: view();
}
```

`scroll()` 預設綁最近的 scroll 祖先。如果中間有不該被計入的 scroll container（例如 sidebar）但你要綁的是 document，用 `scroll(root block)`：`root` 指 document scrolling element，`block` 指 block axis（垂直）。

## `animation-range` — 把 timeline scale 到一段範圍

預設 `view()` 用 `cover` range：元素第一個 pixel 進入 viewport = 0%、最後一個 pixel 離開 = 100%。但很多場景要更精準：

```css
/* 元素剛進場那段做完動畫 */
animation-range: entry 0% entry 100%;

/* 元素已經完全在 viewport 內那段做動畫 */
animation-range: contain 0% contain 100%;

/* "buffer" — entry 100% 應該結束、但讓動畫慢半拍到 entry 150%
   意思是「動畫在元素進場 + 一個 viewport 高度的 50% 額外捲動後才完成」 */
animation-range: entry 0% entry 150%;
```

四個 range keyword 分別對應 view timeline 的不同階段：

- `cover`：first pixel in → last pixel out（完整 range）
- `contain`：fully visible 那段（小元素可能根本沒有 contain range）
- `entry`：first pixel in → fully visible
- `exit`：starting to leave → last pixel out

百分比可以超過 100% 或低於 0% 把 range 「擴出去」做 buffer 效果。`entry 150%` 不是「entry 之後」— 是把 entry 的自然 100% 點當參考、再多走 50% 的距離。

## `animation-fill-mode: both` 防 flicker

這個是 view-timeline 最常踩的坑。預設 `fill-mode: none`，意思是 keyframe 在 range 之外不生效 — 元素回到 CSS declaration 的「靜態」狀態。

實務上看起來：

```css
.card {
  opacity: 0;                    /* 靜態 */
  animation: fade-in linear;     /* fill-mode: none */
  animation-timeline: view();
  animation-range: entry;
}
@keyframes fade-in {
  to { opacity: 1; }
}
```

捲動時序：
1. 元素還沒進場 → range 之前 → keyframe 不生效 → opacity 跳回靜態的 `0` ✓
2. 元素進場中 → range 內 → keyframe 生效 → opacity 從 0 動到 1 ✓
3. 元素 fully in → 過了 entry range → keyframe 不生效 → **opacity 跳回靜態的 `0`** ✗ flicker

修法：`animation-fill-mode: both`（或寫在 shorthand `animation: fade-in linear both`）：

- `forwards` — range 結束後維持 keyframe 終態（解決 step 3）
- `backwards` — range 開始前維持 keyframe 起始（解決 step 1 如果你改寫初始 state 而不是依賴靜態值）
- `both` — 兩邊都管。最 robust，建議預設這樣寫。

## Staggered words — inline 元素的 view() 陷阱

直覺上把 sentence 拆成 `<span class="word">` 每個 word 套不同 `animation-range` 就好。實際上會發現所有 word **同時**動，看起來像整段一起 fade。

原因：inline 元素的 `view()` 抓的是**最近的 block 容器**的 view progress，不是自己的。每個 `<span>` 拿到的 view progress 都是同一個 `<p>` 的進度 → 所有 word 共享 timeline → range 沒意義。

修法：`display: inline-block` 強制 word 變成獨立的 box，view-timeline 才會綁到 word 自己：

```css
.word {
  display: inline-block;     /* 關鍵 */
  opacity: 0;
  animation: fade-up linear both;
  animation-timeline: view();
}
.word:nth-child(1) { animation-range: cover 10% cover 25%; }
.word:nth-child(2) { animation-range: cover 15% cover 30%; }
/* ...每個 word 把 range 往後 offset 5%，做出 stagger */
```

## Linked timelines — 用一顆元素的 view progress 驅動另一顆的動畫

到目前為止 timeline 都綁在「動畫元素自己」或「自己的 scroll 祖先」上。命名 timeline 把 timeline 抽成可以跨元素引用的「變數」：

```css
.scroller {
  /* 在 ancestor 上「保留」這個變數名，讓 sibling/cousin 也能引用 */
  timeline-scope: --content;
}

.paragraph {
  /* 把自己的 view progress 暴露為 --content timeline */
  view-timeline: --content;
}

.square {
  /* 引用 --content — 這顆 square 的動畫進度 = .paragraph 的 view progress */
  animation: morph linear both;
  animation-timeline: --content;
  animation-range: contain;
}
```

兩個關鍵字：

- **`view-timeline: --name`** — 在某顆元素上「發布」一個命名 view-timeline。等同於匿名 `view()` 但有名字、可以被跨元素引用。
- **`timeline-scope: --name`** — 在祖先上「reserve」這個變數名。沒它的話 `--name` 只在「發布它的元素的 sibling」之間可見；有它才能讓「祖先底下任意子孫」都引用得到。

## Multiple animations on the same element

一顆元素可以同時跑多個 scroll-driven 動畫，timeline 跟 range 用 comma 平行寫：

```css
.hero {
  animation: fade-in linear both, slide-in linear both;
  animation-timeline: view(), scroll();
  animation-range: entry 0% entry 50%, cover;
}
```

第一個動畫綁 view-timeline + entry range、第二個綁 scroll-timeline + 整個 cover。每個 animation 的 timeline、range、duration 是獨立的 list — 順序對齊就好。

## Browser support 跟 polyfill

- **Chrome / Edge**：完整支援（115+）。
- **Safari**：基本支援（17.4+）；命名 timeline 還在跟進。
- **Firefox**：還沒。flag 後面有實驗實作。

[scroll-timeline-polyfill](https://github.com/flackr/scroll-timeline) 補上 Firefox。但 polyfill 注意事項：

- 需要 JS 介入 → bundle size 增加（壓縮後仍有 ~10KB）
- 部分 advanced features 不支援或行為不一致（早期版本的命名 timeline）
- 進場前所有 element 在 first paint 跑一次 reflow 計算 timeline，scroll-heavy page 進場可能有 jank

scroll-driven animation 多半是 progressive enhancement — Firefox user 看不到動畫但內容仍可讀，通常可接受。要強上 polyfill 就 ship 它、要省 bundle 就 fallback to static、看情境。

## Gotchas

- **`fill-mode: both` 幾乎是預設**。沒寫 `forwards` 或 `both`，元素在 range 結束後會跳回 CSS 靜態值 → 整段 scroll 看起來閃一下。`forwards` 解決尾端 flicker、`both` 連同前端也一起 cover。
- **Inline 元素的 `view()` 綁的是父容器**。staggered words 必須 `display: inline-block` 才會有自己的 view-timeline。同樣坑也適用 `<a>`、`<em>`、任何沒被 block-ify 的 inline。
- **`timeline-scope` 必須掛在 publisher 跟 consumer 的共同祖先上**。如果 `view-timeline: --content` 在 sibling、`animation-timeline: --content` 在另一個 sibling，它們的「共同 parent」要寫 `timeline-scope: --content`。漏掉的話 consumer 抓不到 timeline、動畫整個不跑。
- **`entry 150%` 不是「entry 之後」**。Range keyword 後面的百分比是「相對該 range 的 0–100% 區間」線性外插。`entry 150%` = 在 entry 結束後再多走半個 entry-range 距離。要做 buffer 用、不要當「進場後」誤解。
- **多動畫的 timeline / range 用 comma 平行對齊**。`animation-timeline: view(), scroll();` 跟 `animation-range: entry, cover;` 跟 `animation: a, b;` 三個 list 順序要對。錯位會讓動畫綁到錯的 timeline。
- **Polyfill 不是 100% 等價**。命名 timeline、timeline-scope、複雜 range 在 polyfill 上可能行為不一致。production 用 polyfill 一定要在 Firefox 跑過實機 + 標 known limitations。
- **不要把關鍵內容只放在 scroll animation 內**。scroll-jacking + 沒 scroll 就看不到 = a11y 災難。`prefers-reduced-motion: reduce` 一定要 fallback 成靜態狀態（demo CSS 內已加）。視覺增強 OK、阻擋資訊不行。
- **`scroll(root block)` 越過中間 scroll containers**。預設 `scroll()` 綁最近 scroll 祖先；page 內有 sidebar / modal 等 nested scroll container 時、想綁 document scroll 要明寫 `scroll(root block)` — `root` 跳到 document、`block` 指垂直軸。
