## TL;DR

- `document.startViewTransition(callback)`：browser 拍照舊 UI → 跑 callback（你做 DOM 突變）→ 拍照新 UI → 在最上層 pseudo-element 上 tween 兩張 snapshot → 拆掉 pseudo、露出真 DOM。預設行為：整頁 cross-fade。
- `view-transition-name: foo` 給某個元素自己的 transition group → 它跟其他元素獨立 tween。**list reorder** 的關鍵：每個 item 有自己的 name，browser 對位置變動自動產生 `transform: matrix(...)` keyframes 從舊 bounding box 滑到新 bounding box。
- 自訂 keyframes 用 `::view-transition-old(name)` / `::view-transition-new(name)` 蓋掉預設 cross-fade。`view-transition-class: foo` 給多個 elements 同 class、再用 `::view-transition-group(.foo)` 一次套樣式，不用每個 name 寫一次。
- `view-transition-name: match-element` 是讓 browser 自動配 unique name 的 keyword — 配上 `view-transition-class` 就能「自動命名 + 共用樣式」。
- **React 大坑：`flushSync` 必須包 setter**。state 預設 async，VT callback 跑完還沒 commit → browser snapshot 到舊 UI → 沒得 tween。`flushSync(() => setX(...))` 強迫 React 同步 commit。
- Browser support ~85-90%；feature detect `if ('startViewTransition' in document)` fallback 直接 mutate。`prefers-reduced-motion` 也走 fallback path。

## API 三步驟

```ts
document.startViewTransition(() => {
  // 你的 DOM mutation（換 route、reorder list、unmount/mount...）
  setItems(shuffled);
});
```

實際發生的事：

1. **Snapshot old**：browser 把當前 viewport 內容拍成 `::view-transition-old` pseudo-element（每個有 `view-transition-name` 的元素各拍一張，其餘整片合成一張 root snapshot）。
2. **Run callback**：跑你的 mutation。callback 結束 = DOM 已是新狀態。
3. **Snapshot new**：browser 拍新狀態 → `::view-transition-new`。
4. **Animate**：在頁面最上層覆蓋一層 pseudo tree，在 old / new pseudo 之間 tween。預設 250ms cross-fade，可被自訂 keyframes 蓋掉。
5. **Teardown**：tween 結束移除 pseudo tree，露出真 DOM。

過程中**整頁不可互動** — click 點不到任何東西。也因為 pseudo tree 蓋在最上層，期間做更多 DOM 變動不會反映在 snapshot 上。

## Pseudo-element 結構

每個 `view-transition-name` 開出一棵 pseudo tree：

```
::view-transition                          ← 最外層 root
└─ ::view-transition-group(name)           ← 控制 group 整體（size、position transition）
   └─ ::view-transition-image-pair(name)   ← 包住 old + new
      ├─ ::view-transition-old(name)       ← 舊 snapshot（image）
      └─ ::view-transition-new(name)       ← 新 snapshot（image）
```

- 想改整體 transition 的 timing / curve → 改 `::view-transition-group(name)`。
- 想自訂進 / 出場 keyframes → 改 `::view-transition-old(name)` / `::view-transition-new(name)`。
- 沒給 name 的元素全部塞進 `::view-transition-old(root)` / `::view-transition-new(root)`，整個 cross-fade。

## `view-transition-name` 的 list reorder 魔法

```css
.tile {
  view-transition-name: match-element; /* browser 自動配 unique name */
  view-transition-class: tile;          /* 共用 class 給後面選樣式 */
}
::view-transition-group(.tile) {
  animation-duration: 600ms;
  animation-timing-function: cubic-bezier(0.65, 0, 0.35, 1);
}
```

每個 tile 有自己的 name → 各自一棵 pseudo tree。位置從 (x1, y1) 變到 (x2, y2) 時，browser **自動產生** 一段 transform keyframes：

```css
@keyframes (auto-generated) {
  from { transform: matrix(1, 0, 0, 1, dx, dy); }
  to   { transform: matrix(1, 0, 0, 1, 0, 0);   }
}
```

`matrix(a, b, c, d, e, f)` 前 4 個值是 scale / rotate / skew，最後 2 個 (`e`, `f`) 是 translate x / y。reorder 沒有 scale 變化、純位移 → 前 4 值固定 `1, 0, 0, 1`，後 2 值是「舊 bounding box 中心減新 bounding box 中心」。size 也變了的話 a / d 會放 scale ratio。

不需要你寫任何 keyframes — 純標 name + 觸發 mutation 就有 FLIP-style 動畫，免 `motion.div`、免 `layout` prop。

## `match-element` + `view-transition-class`

寫死 `view-transition-name: tile-1`、`tile-2`...... 對 16 個 tile 太累。`match-element` keyword 讓 browser 替每個 element 自動配 unique name。

但這樣每個 name 不同 → 沒辦法用 `::view-transition-group(name)` 一次選到全部。修法：再給一個 `view-transition-class: tile`，CSS 用 class selector：

```css
::view-transition-group(.tile) { animation-duration: 600ms; }
```

「自動配名」+「共用樣式」兩全。

## 自訂 keyframes：slideshow

預設 cross-fade 對 list reorder 夠用，但 slideshow 想要橫向滑入滑出。把同一個 element 的 view-transition-name 兩次 snapshot 接起來改成自訂 keyframes：

<DemoFrame title="Slideshow — 自訂 enter/exit keyframes">
  <VtSlideshow />
</DemoFrame>

```css
.slide {
  view-transition-name: slideshow;
}
::view-transition-old(slideshow) {
  animation: 450ms ease both exit-left;
}
::view-transition-new(slideshow) {
  animation: 450ms ease both enter-right;
}
@keyframes exit-left  { to   { transform: translateX(-100%); opacity: 0; } }
@keyframes enter-right { from { transform: translateX(100%);  opacity: 0; } }
```

按 Next：browser 拍舊 slide → callback `setIndex(next)` → 拍新 slide → 跑你寫的 keyframes（舊的滑出左、新的滑入右）→ teardown。沒有真的「兩個 slide 並排再 translate」的 DOM 操作 — 純 CSS pseudo。

**雙向**：按 Prev 想反方向滑，給 `<html>` 加一個 `vt-slideshow-back` class、CSS 用 descendant selector 換掉 keyframes：

```css
.vt-slideshow-back ::view-transition-old(slideshow) { animation-name: exit-right; }
.vt-slideshow-back ::view-transition-new(slideshow) { animation-name: enter-left; }
```

`::view-transition-*` pseudo-elements 掛在 document root 下，所以 `<html>` 的 class 影響得到。tween 結束 (`transition.finished` resolve) 後再把 class 拔掉。

## React 跟 `flushSync` 的關係

```tsx
// ❌ 不會動 — VT 拍到舊 UI 兩次
document.startViewTransition(() => {
  setWords(shuffle(words));
});
```

React 18+ state 是 async — `setWords` 把 update 排進 queue，**callback 結束時還沒 commit**。VT 拍 snapshot 看到的還是舊 DOM = 兩張 snapshot 一樣 = no-op tween。

```tsx
// ✅ flushSync 強迫同步 commit
import { flushSync } from 'react-dom';

document.startViewTransition(() => {
  flushSync(() => setWords(shuffle(words)));
});
```

`flushSync` 是 react-dom 出來的逃生艙 — 強迫 React 在當下這個 microtask 內 commit 完成 update。callback 結束時 DOM 已是新狀態 = VT 拍到正確的後 snapshot。

代價：放棄 batching、放棄 concurrent rendering 的優化。但 `startViewTransition` 一秒最多觸發幾次（user 操作驅動），跑 sync 對效能無感。

`flushSync` 不是 hack — 就是給「需要在某個 sync 邊界後 DOM 已經更新」的場景用的。VT、`requestAnimationFrame`、focus management 都會踩到。

## Feature detect + reduced motion

```ts
function applyChange() {
  setWords(shuffle(words));
}

if (
  !('startViewTransition' in document) ||
  window.matchMedia('(prefers-reduced-motion: reduce)').matches
) {
  applyChange(); // 直接 mutate，沒動畫
  return;
}

document.startViewTransition(() => {
  flushSync(applyChange);
});
```

兩個 fallback 條件：

1. **不支援 VT**：~85-90% browser support，剩下的（舊 Safari / Firefox 沒開 flag）走 plain mutate path。注意是 `'startViewTransition' in document` 而不是 `if (document.startViewTransition)` — TS 對後者會報「Property does not exist」如果 lib.dom 沒最新版。
2. **prefers-reduced-motion**：user OS 設定 reduce motion → 不該套 VT。直接 mutate 是正確 fallback（user 看到瞬切是預期行為）。

## Gotchas

- **`flushSync` 必填 inside callback**。React state 是 async，沒 flushSync = VT 拍兩張一樣的 snapshot = no-op。錯誤訊號：「`startViewTransition` 跑了但畫面瞬切」。
- **Feature detect 用 `'startViewTransition' in document`**。寫 `if (document.startViewTransition)` 在 TypeScript 舊 lib.dom 環境下會報 type error — `in` 是 type-narrowing 友善的寫法。
- **`prefers-reduced-motion` 走 fallback path**。OS 設定 reduce motion 的 user 不該被強塞 VT，直接 mutate 即可（user 預期動畫被關掉就是瞬切）。
- **Page 在 VT 期間不可互動**。click、keydown 全被 ignore（pseudo tree 蓋在最上層 + non-interactive）。VT duration 別設超過 1s，否則感覺像卡住。要做「中斷」(skip transition、立刻換新) 是另一課題（browser API 是 `transition.skipTransition()`）。
- **自訂 keyframes 必須給 `animation-duration`**。`::view-transition-old(name) { animation-name: foo; }` 只指定 name 沒給 duration → fallback 到 default 250ms，感覺常常「跟想像不同」。寫 shorthand `animation: 600ms ease both foo;` 一次給齊。
- **沒給 `view-transition-name` 的兄弟元素在 VT 期間會「消失」**。它們被塞進 root group cross-fade，但若它們是某個 named group 的兄弟、在 named group 動的同時，root cross-fade 的 timing 不一定跟你想的一樣。修法：給它 `view-transition-name` （或就讓它整個塞進 root group 接受 cross-fade 行為）。
- **`match-element` 自動配名是 anonymous — 沒辦法用 `::view-transition-group(name)` 直接選**。必須搭 `view-transition-class: foo` 才能用 `::view-transition-group(.foo)` 選到。光 `match-element` 沒 class 等於只給了 transition group、無法自訂樣式。
- **自訂 keyframes 不能寫在 inline `style={...}`**。pseudo-element 規則必須在 stylesheet 裡（`<style>` block / external CSS / shadow DOM）。inline style 套不到 `::view-transition-*`。
- **跨頁面 VT 走另一個 API**。SPA 內部 mutate 用 `startViewTransition`；MPA 跨 navigation 要 `@view-transition { navigation: auto; }` 或 navigation API + page-reveal event。本課只覆蓋 SPA case。
- **同 page 多個 demo 共用 `view-transition-name` 會撞**。VT name 是 document-global — 兩個元件同時掛同一個 name = ambiguous，browser 通常 fallback 到沒名字的 root cross-fade。長頁面 / 多 widget 用 `useId()` 之類做 namespace。
