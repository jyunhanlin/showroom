## TL;DR

- `Math.sin(t)` 永遠在 `[-1, 1]` 之間，而且是平滑曲線 — 不需要 cubic-bezier 就有 ease-in-out 的感覺。
- `normalize(Math.sin(t), -1, 1, lo, hi)` 把振盪映射到 canvas 任意座標範圍。
- `sin` for x + `cos` for y = 圓周;5 顆球 + `index * offset` 串成 loading spinner;x/y 給不同頻率 = Lissajous 蟲形。

## Why sin/cos trace a circle

幾何上,`(cos θ, sin θ)` 隨 θ 從 0 到 2π,軌跡剛好就是單位圓上的一點。把 θ 換成 `time`(秒)、各自 `normalize` 到 canvas 座標範圍就完事:

```ts
const totalTime = (now - startTime) / 1000;

const x = normalize(
  Math.sin(totalTime * 2),
  -1, 1,
  canvasDimensions.width * 0.2,
  canvasDimensions.width * 0.8,
);
const y = normalize(
  Math.cos(totalTime * 2),
  -1, 1,
  canvasDimensions.width * 0.4,
  canvasDimensions.width * 0.6,
);
```

`sin` 跟 `cos` 是相位差 90°的雙胞胎 — x 達到極值時 y 在中央,反過來也成立。把這兩個值同時當 (x, y) 用,軌跡就閉合成圓 (或橢圓,看 x、y 的 normalize 範圍是否對稱)。

`* 2` 是頻率倍數。`Math.sin(t)` 的週期是 2π 秒(JS 三角函式吃 radian),乘 2 變成 π 秒一圈。想要剛好 1 秒一圈就 `* Math.PI * 2`。

## 5 balls + offset = loading spinner

每顆球差一點點時間相位:

```ts
balls.forEach(({ color }, index) => {
  const offset = index * 0.3;
  const x = normalize(Math.sin(totalTime * 2 + offset), -1, 1, ...);
  const y = normalize(Math.cos(totalTime * 2 + offset), -1, 1, ...);
  // ...
});
```

`Math.sin(t + 0.3)` 等於 `Math.sin(t)` 提前 0.3 秒的版本。每顆球都是「下一顆過 0.3 秒會在的位置」,視覺上就是後一顆追前一顆,5 顆排成一條跟隨蛇 — 這是 loading spinner 最常見的長相。

## Asymmetric frequencies → Lissajous worm

x 跟 y 不再共用同一個 speed,軌跡就不再閉合成圓:

```ts
const xSpeedMultiple = 1.5;
const ySpeedMultiple = xSpeedMultiple * 5;
const x = normalize(Math.sin(totalTime * xSpeedMultiple + xOffset), -1, 1, ...);
const y = normalize(Math.cos(totalTime * ySpeedMultiple + yOffset), -1, 1, ...);
```

x 一秒 1.5 圈,y 一秒 7.5 圈 — y 軸劇烈震盪同時,x 軸慢慢掃過去。視覺上是條蟲在水裡扭動。

數學上叫 [Lissajous curves](https://en.wikipedia.org/wiki/Lissajous_curve)。x:y 頻率比是有理數時軌跡會閉合 (跑久了會重複);非有理數比會永遠不重複。1.5:7.5 = 1:5,所以這個 demo 是會閉合的。

## Gotchas

- **y 範圍是 `width * ...`,不是 `height * ...`** — 課程刻意這樣寫,讓 orbit 的形狀以 width 為基準,不會被 canvas aspect 拉扁。代價:canvas 高度必須 ≥ `width * 0.65 + BALL_RADIUS` (worm),否則球底會被截掉。Demo 用 `w-[400px] h-[300px]`,worm 球最低點 = `400 * 0.65 + 25 = 285 < 300`。換成 `w-full` 會在寬螢幕上完全看不到球。
- **`Math.sin(time)` 不需要 deltaTime**。它本身就是時間的純函數,frame rate 變動只影響採樣密度,不影響軌跡形狀。 `Math.sin(0.5) === Math.sin(0.5)` 永遠成立。比起 `box.x += velocity * deltaTime`,sine motion 完全不會因為 frame skip 而 drift,也不需要 250ms clamp。
- **`startTime` 要在 `useEffect` 裡記**。Strict Mode double-mount 時兩次掛載會有不同的 mount 時刻;`startTime = performance.now()` 寫在 useEffect 內,每次 mount 重新 anchor,初始相位永遠是 0。寫成 module-level 兩次共用,第二次掛載會看到「相位偏移」的詭異初始位置。
- **`normalize` 的 5-arg 簽名**。`src/utils/canvas.ts` 的 `normalize(value, inMin, inMax, outMin, outMax)` 跟課程一致;3-arg 呼叫 `normalize(value, min, max)` 沿用預設值 `outMin = 0, outMax = 1`,等同舊行為,不破壞 `clampedNormalize` 的內部使用。
