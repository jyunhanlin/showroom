## TL;DR

- Bezier 是單峰：起點、(可能) 一個鼓包、終點，progress 永遠 0 → 1 不回頭。能模仿 ease-in / ease-out / ease-in-out / ease，但沒辦法做 overshoot + 衰減震盪。
- Spring physics 模擬「彈簧連接 + 阻尼 + 重力」的數學模型，自然產生 overshoot 跟逐漸衰減的回彈。心跳放大、push notification 進場、彈跳清單這類動畫用 spring 才有「物理感」。
- 以前實作 spring：JS library（React Spring、Motion）逐 frame 算物理位置，效能比 CSS 差因為走 main thread。
- **CSS `linear()` 函式（2023 開始所有主流瀏覽器支援，~92%）** 可以把 spring physics 算過的離散點「baked」進 timing function，CSS engine 自己 interpolate — 等於 spring physics 進到 compositor thread。
- 這節是 module 7 (springs) 的 intro，後面 3 節 (Intro to Springs / linear() / Limitations) 才是 meat。

## Bezier 為何撞牆

CSS easing 預設值都是 cubic Bezier：

- `linear` — 直線（其實就是 `cubic-bezier(0, 0, 1, 1)`）
- `ease`、`ease-in`、`ease-out`、`ease-in-out` — 各種曲線形狀
- 自訂 `cubic-bezier(0.42, 0, 0.58, 1)` — 4 個 control point 數字定義

Bezier 的數學保證 progress **單調遞增**（不回頭、不超過 1）。所以：

- ✅ Ease-in（緩慢起步加速到底）
- ✅ Ease-out（快起步緩慢到底）
- ✅ Ease-in-out（兩端慢中間快）
- ❌ Overshoot 110% 再回到 100%
- ❌ 來回震盪幾次再停
- ❌ 一開始倒退再前進

最後三個都是 spring 的招牌行為，Bezier 不行。

## Spring 是什麼

物理上的彈簧連在固定點跟運動物體之間。鬆開時：

1. 彈簧拉動物體到平衡點
2. 物體**衝過**平衡點（慣性）
3. 反向被拉回
4. 阻尼吸收能量，震盪幅度逐漸縮小
5. 最後停在平衡點

數學模型有三個參數：

- **stiffness** (緊度) — 彈簧多硬。越硬震盪越快，但要更久衰減
- **damping** (阻尼) — 環境多濃。低 damping = 真空，永遠震盪不停；高 damping = 糖漿，沒震盪直接到位
- **mass** (質量) — 物體多重。Josh 說這個基本上不用調

不需要 duration — 動畫多長**取決於物理**（spring 多久才完全停下來）。

## linear() 是新出口

```css
.elem {
  transition: transform 500ms linear(0, 0.1, 0.25, 0.5, 0.68, 0.8, 0.88, 0.94, 0.98, 0.995, 1);
}
```

`linear()` 接 N 個 0~1 之間的數字（也可以超出，1.25 = overshoot 25%）。瀏覽器把它們等距放在 timeline 上，用直線連接 = polyline timing function。

雖然每段是直線，但點夠密就近似任何曲線。Spring physics 的離散採樣丟進去就重現 spring。

## Gotchas

- **`linear` 跟 `linear()` 是兩個不同東西**：
  - `linear` (沒括弧) = `cubic-bezier(0, 0, 1, 1)` 的 alias，恆速
  - `linear()` (有括弧 + 引數) = polyline timing function，可以模擬任何曲線
- **`linear()` 92% 支援（2026/04）** — 主流瀏覽器全通，少數舊版 Safari 沒有。Fallback 通常給 `ease` 即可，動畫退化成普通 cubic。
- **bezier 沒過時** — ease-in / ease-out 這類「不需要 overshoot」的場合 bezier 仍然合適，比 linear() 字串簡潔。Spring 是補位、不是取代。
