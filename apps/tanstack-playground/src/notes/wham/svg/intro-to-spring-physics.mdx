## TL;DR

- **Stiffness** (= tension)：彈簧多緊。**高 stiffness → 震盪頻率快 + 過衝大**；低 stiffness → 平滑 glide 沒震盪。
- **Damping** (= friction)：環境多黏。**Damping=0 → 真空 → 永遠震盪**；高 damping → 糖漿，幾乎沒過衝直接到位。
- **Mass**：物體質量。Comeau 說「我從不調這個」 — 1 永遠 ok，調 stiffness/damping 比較直觀。
- **沒有 duration**：spring 動畫多長是物理結果，spring settle 下來才結束。Time-based timing function (CSS transition `1000ms`) 跟 spring 概念衝突，linear() 那節再講怎麼兜。
- **Tweak 流程**：先把 damping 設 20（中性）→ 調 stiffness 對「速度感」→ 再調 damping 對「震盪 vibe」。

## Stiffness — 緊度 = 速度 + 震盪

```js
animate(elem, { x: 240 }, { type: 'spring', stiffness: 800, damping: 20 });
```

物理直覺：彈簧越硬，儲存的能量越多。物體被快速拉到平衡點 → **能量太多停不下來** → 過衝 → 反彈 → 震盪幾下耗能 → 才停。

- `stiffness: 50` — 鬆軟、慢速、幾乎不震盪 → 像 ease-out 但更平順
- `stiffness: 200`（預設）— 中等速度，可能輕微 overshoot
- `stiffness: 800` — 緊硬、超快、明顯彈跳 → boingy-boingy

**關鍵 insight**：低 stiffness 出來的動畫**很像精心調過的 cubic-bezier ease**，但設計上更輕鬆 — Bezier 要找對 4 個 control point 才好看，spring 設個 50 / 20 就出來。

## Damping — 阻尼 = 衰減

```js
animate(elem, { x: 240 }, { type: 'spring', stiffness: 200, damping: 0 });
```

把 damping 設 0：物體在真空中，沒摩擦力。Spring 把物體拉到平衡點，物體衝過去→反彈→沒能量損失→繼續震盪→**永遠不停**。

(實作上 motion library 通常還是會在某個振幅閾值切斷動畫，但理論上是無限。)

- `damping: 0` — 永動機（不實用）
- `damping: 5` — 大量震盪，緩慢衰減
- `damping: 20`（預設）— 1-2 次小過衝後 settle
- `damping: 50` — 幾乎沒過衝，平滑到位

## Mass 不用調

Mass 影響「同 stiffness 下物體加速多快」 — 但這跟 stiffness 變化等價，調 stiffness 涵蓋同樣 design space。

Comeau 提到 mass 純粹是 completeness（library 都有這 param）。**保持 mass=1 一輩子都不會踩到坑**。

## 沒有 duration 的反直覺

```css
/* CSS transition — 必須有 duration */
.elem { transition: transform 500ms ease-out; }

/* Spring — 自然發生 */
animate(elem, { x: 240 }, { type: 'spring', stiffness: 200, damping: 20 });
```

CSS transition 必須有時間：「這動作走 500ms」。Spring 沒這選項。Settle 時間是 stiffness × damping 的數學結果，可能 200ms 也可能 2 秒。

**這就是為什麼 spring 進到 CSS 是個棘手問題** — CSS transition 模型骨子裡是時間驅動。`linear()` 的解法是「事先算好 spring physics 的離散採樣，硬塞進固定 duration」，下兩節會講 tradeoff。

## Tweak 流程

1. **stiffness 調速度感**：先看動畫從 A 到 B 大概多快算合理（200ms？800ms？），調 stiffness 直到速度對。先別管 bouncy。
2. **damping 調 vibe**：速度 OK 後，
   - 想要彈跳生動 → damping 低（5-15）
   - 想要平滑優雅 → damping 高（30-50）
   - 不確定 → damping=20 中性
3. **互相干擾**：stiffness 調很高後 damping 也要跟著調高，否則震盪太久。

## Stiffness/damping vs Tension/friction

兩組命名是同義詞：

| 概念 | 詞 A | 詞 B | Library |
|---|---|---|---|
| 緊度 | stiffness | tension | Motion / Easing Wizard 用 stiffness |
| 阻尼 | damping | friction | React Spring 用 tension / friction |

切換 library 時注意換詞，但概念完全一致。

## Gotchas

- **Demo 用 `animate()` from 'motion'** — Motion library 的 imperative API 包了 spring physics 計算。直接傳 `{ type: 'spring', stiffness, damping }` 即可。
- **Strict Mode double-invoke 沒問題**：每次 toggle 都 `animate()` 重 trigger，前一個動畫被 cancel。React 開發環境 double-mount 也只是動畫被 cancel 重跑而已。
- **interrupt（動畫沒跑完就 trigger 第二次）的行為各 library 不同**：React Spring 保留當前 inertia 平滑接續，Motion 直接重新 spring（看起來 OK 但不是「物理正確」），CSS linear() 用 reverse shortening factor（速度感變奇怪）。Limitations 那課會詳解。
- **`type: 'spring'` 是 motion 的特殊 timing**，不是 CSS spec 內建。要寫純 CSS spring 必須用 linear() 函式（下節）。
